# optimizationmaxxing — Professional Audit (2026-06-18)

Four parallel research streams (internal inventory, competitor parity, SOTA tweak research, coverage/UX gaps) run against v0.2.5. This file is the durable record + source receipts. Shipped fixes landed in **v0.2.6** (catalog v1.8.6).

## Verdict

Above competitor baseline on philosophy and depth (reversible-only, honest scope, real diagnostics) and already shipping things competitors charge for. Not yet "the pinnacle" for three reasons: (1) a few shipped tweaks had aged into the "revert to default" camp; (2) NVIDIA-desktop-centric coverage; (3) a few missing free wins + trust/automation features competitors monetize.

## What we already do well (don't rebuild)

- One-click optimize → **TuneNow** (`/tune`). Before/after benchmark + shareable card → **Asta Bench**. Clean revert → snapshot store + Restore Point.
- We already ship the **biggest hidden tax**: `vbs.hvci.disable` + `bcd.hypervisorlaunchtype.off` (VBS/Memory Integrity, default-on in 24H2, 5–15% CPU-bound).
- `process.hags.enable` is correct per 2026 (HAGS ON is required infra for Reflex/Anti-Lag 2).
- We disable Game DVR but leave Game Mode on (correct). EPP/mouse-accel already covered by `ui.mouse.disable-acceleration` (MouseSpeed/Threshold=0) — the "missing EPP" audit finding was a false positive.

## Shipped in v0.2.6 (catalog v1.8.6)

1. **`bcd.disabledynamictick.yes`** — Microsoft debug-only flag; 2026 testing ties it to Win11 mouse-input desync + added latency. Rewrote copy, gated to Win10 (`osMaxBuild: 19045`), removed from Frame Pacing + Asta presets. Kept reversible for Win10 tuners.
2. **`net.throttling.disable`** (NetworkThrottlingIndex) — best-instrumented testing shows no FPS/latency gain vs default 10, sometimes worse. Removed the false "frees the pipe" claim, de-recommended, removed from BR / Network-Low-Latency / Asta presets. Kept available.
3. **`monitor.mpo.disable`** — reframed as a conditional flicker fix, NOT an FPS lever (disabling can cost the Independent Flip low-latency path on healthy single-monitor rigs). Added `OverlayMinFPS=0` for 24H2 reliability.
4. **NEW `power.fast-startup.disable`** (`HiberbootEnabled=0`) — disables Fast Startup while keeping hibernation (unlike `powercfg /hibernate off`). Clean cold boot, fixes stale-state stutter / post-update glitches / dual-boot clock drift. Risk 1, free, reversible.
5. **NEW guide `snake-oil-tweaks`** — "Tweaks we deliberately don't do," surfaced on `/guides` (badge NO SNAKE-OIL).

## Staged backlog (NOT shipped in v0.2.6) + gate rationale

User instruction was "do them all if 100% beneficial." These were held:

**Failed the "100% beneficial / can do correctly now" gate — need a design decision, would violate no-snake-oil/no-speculation if rushed:**
- **DPC-latency culprit attribution** — accurate attribution needs ETW/xperf tracing; a naive "iCUE is probably your spike" is exactly the speculation the product forbids.
- **Silent drift auto-heal + per-game launch hook** — both require an always-running background process that hooks game launches; for an anti-cheat-adjacent tool that's an optics/safety tradeoff, not a pure win. Should be opt-in re-offer, not silent. Needs Diggy's call.
- **Audio format/enhancement auto-writes** — per-endpoint MMDevices PropertyStore writes are unreliable across devices; would risk shipping a tweak that silently no-ops. Belongs as a guide.

**Pass the gate but are large content/feature builds (next focused batches, not a rushed pass):**
- **GPU-vendor fork** — Radeon (Adrenalin: Anti-Lag 2 / RSR / HYPR-RX) + Intel Arc (driver-currency + XeSS) guidance + live telemetry mirroring the NVIDIA path. Biggest coverage hole (live GPU telemetry is nvidia-smi only; GPU guidance is NVPI only).
- **Intel APO / AMD X3D-parking health checks** — detection-driven "is your best free first-party lever on?" advisories.
- **Native Windows System Restore point before apply** (`Checkpoint-Computer`) — distinct from our SQLite snapshot; strongest "won't brick your rig" signal. Caveat: System Protection is often off by default + 24h rate-limit, so needs an enable-protection flow (design decision: do we consume disk to force-enable?).
- **Drift auto-heal + post-Windows-update monitor** (opt-in) — `/diff` detects drift but doesn't re-offer/heal; undercuts Paragon's paid $25 checkup.
- **OS-build-aware gating** — schema supports `osMinBuild`/`osMaxBuild`; apply to build-divergent tweaks (Win10 EOL'd 2025-10-14; 23H2 vs 24H2).
- **Thin-game coverage** — Warzone / OW2 / osu! / Marvel Rivals get only process-suspend; no configs/pro-settings. Grind is 11/11 Fortnite. Needs current per-game pro-config research (volatile — verify before shipping).
- **Consider next:** re-label `monitor.windowed-game-opt.disable` — disabling Win11 windowed-game optimizations removes the flip-model latency benefit for borderless players; currently labeled "benefit on some, stutter on others" (acceptable but could be sharper).
