## A. Catalog (`resources/catalog/v1.json`) — correctness

**A1. BROKEN/INERT — `monitor.windowed-game-opt.disable`** (ship first, it's a dead tweak)
- Writes `DirectXUserGlobalSettings = "GlobalOptimizationsFlags=0;"` — that token is unrecognized, Windows ignores it, the optimization stays ON. Inert.
- Correct token: `"VRROptimizeEnable=0;SwapEffectUpgradeEnable=0;"`.
- THEN reframe: on 24H2 (osMinBuild 26100) this opt IS the low-latency DirectFlip/auto-HDR path; disabling can COST latency on a healthy single-monitor rig. Re-class as conditional stutter-fix like `monitor.mpo.disable`, not a blanket win.
- Src: MS Support "Optimizations for windowed games in Windows 11"; Winhance #363; gaming_os_tweaker gpu.cmd.

**A2. CONTRADICTS OWN DOC — `ui.gamedvr.disable`**
- 2nd action writes `GameBar\AllowAutoGameMode = 0`, which disables Windows **Game Mode**. Title/rationale never mention Game Mode. `snake-oil-tweaks.md` explicitly says "Game Mode: leave ON." 
- Fix: drop the `AllowAutoGameMode=0` action; keep `GameDVR_Enabled=0`. Recording is already killed by the GameDVR key + `ui.gamedvr.appcapture.disable`.

**A3. DISPROVEN TIMER PAIR — `process.hpet.disable` + `bcd.useplatformclock.disable`**
- `useplatformclock=no` is already the default on modern CPUs (invariant TSC) → tweak is a no-op, yet ships risk-3/VIP claiming "single most important tweak for 0.1% low stability" (false).
- `process.hpet.disable` (hpet `Start=4`) is worse: TSC periodically resyncs against HPET; remove HPET → clock drift over long sessions → can be mistaken for a speedhack by some anti-cheats.
- Contradicts own `snake-oil-tweaks.md` ("bcdedit clock flags: revert to default").
- Fix: same treatment as `bcd.disabledynamictick.yes` — remove from presets, mark optional/contested, add clock-skew→speedhack note to hpet (beyond existing battleye risk tag).
- Src: Melody's Tweaks "Misconceptions about timers"; Blur Busters; ROG forum.

**A4. MISSING FORM-FACTOR GATE — `process.connected-standby.disable`** (`PlatformAoAcOverride=0`)
- 2023+ Modern-Standby-only laptops often have no S3 fallback; forcing AoAc off can break clean sleep. Sibling power tweaks already carry `formFactor:["desktop"]`.
- Fix: add `formFactor:["desktop"]` or a laptop warning. (lower confidence)

**A5. OVERSTATED COPY — NIC offload cluster** (`net.nic.interrupt-moderation.disable` "single highest-impact NIC latency win", `checksum-offload`, `lso`, `rsc`)
- Catalog's own v1.8.7 note concedes most NIC tuning is placebo (ping/jitter ISP-dominated; competitive UDP is small). Soften the strong pro-claims to match the honest `net.throttling.disable` treatment. Not harmful, just over-promising. (`net.nic.eee-powersave.disable` correctly kept as the one real win.)

**A6. HIGH-VALUE MISSING — Communications audio ducking → "Do nothing"** (ADD)
- `HKCU\Software\Microsoft\Multimedia\Audio\UserDuckingPreference` (DWORD) = **3**. Default 0 = reduce other sounds 80% on "communications activity" (Discord/voice) → buries footstep/positional cues. Setting 3 stops it.
- Reversible, HKCU (no UAC), measurable. Suggest free tier, risk 1, audio/process category.
- Src: MS GPP docs; UnattendedWinstall #76; MS Learn threads.

**Verified CORRECT (no change):** `priority-separation=0x26`, `system.responsiveness=0` (MMCSS ducking only >~50), `hags.enable`, MPO reframe, `disabledynamictick` Win10 gate, `net.throttling.disable` de-rec, Spectre/Meltdown DANGER tags, `global-timer-resolution` Win11 gate.
